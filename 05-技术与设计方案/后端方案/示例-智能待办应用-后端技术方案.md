# 后端技术方案：智能待办应用（SmartTodo）

## 1. 技术选型

| 维度 | 选择 | 说明 |
|------|------|------|

## 3. 数据模型设计

### Prisma Schema (核心部分)

```prisma
model User {
  id           String         @id @default(uuid())
  email        String         @unique
  name         String
  passwordHash String         @map("password_hash")
  tasks        Task[]
  preference   UserPreference?
  createdAt    DateTime       @default(now()) @map("created_at")
  updatedAt    DateTime       @updatedAt @map("updated_at")

  @@map("users")
}

model Task {
  id           String       @id @default(uuid())
  userId       String       @map("user_id")
  title        String
  description  String?
  dueDate      DateTime?    @map("due_date")
  priority     Priority     @default(MEDIUM)
  status       TaskStatus   @default(TODO)
  isAiPriority Boolean      @default(false) @map("is_ai_priority")
  user         User         @relation(fields: [userId], references: [id])
  aiAnalysis   AIAnalysis?
  reminders    Reminder[]
  createdAt    DateTime     @default(now()) @map("created_at")
  updatedAt    DateTime     @updatedAt @map("updated_at")

  @@map("tasks")
}

model AIAnalysis {
  id            String   @id @default(uuid())
  taskId        String   @unique @map("task_id")
  priorityScore Float    @map("priority_score")
  reasoning     String
  context       Json?
  task          Task     @relation(fields: [taskId], references: [id])
  analyzedAt    DateTime @default(now()) @map("analyzed_at")

  @@map("ai_analyses")
}

enum Priority {
  HIGH
  MEDIUM
  LOW
}

enum TaskStatus {
  TODO
  IN_PROGRESS
  DONE
}
```

## 4. 服务层设计

| 服务名称 | 职责 | 依赖 | 对应PRD功能 |
|----------|------|------|-------------|
| AuthService | 用户认证（注册/登录/Token管理） | UserService, JWT | 基础能力 |
| UserService | 用户 CRUD、偏好管理 | Prisma | 基础能力 |
| TaskService | 任务 CRUD、列表查询、状态管理 | Prisma, AIService | FR-001, FR-003 |
| AIService | 自然语言解析、优先级分析、今日推荐 | OpenAI API, Redis | FR-001, FR-002, FR-003 |
| NotificationService | 提醒调度、推送发送 | Bull Queue, APNs/FCM | FR-004 |

## 5. 认证与授权

| 项目 | 方案 |
|------|------|
| 认证方式 | JWT (Access Token 15min + Refresh Token 7天) |
| Token 存储 | Access Token: 客户端内存; Refresh Token: HttpOnly Cookie / SecureStore |
| 刷新机制 | `/auth/refresh` 端点，自动换发 Access Token |
| 权限模型 | 简单用户级（用户只能操作自己的数据） |

## 6. 错误处理规范

### 业务错误码

| 错误码 | HTTP 状态码 | 说明 |
|--------|------------|------|
| AUTH_001 | 401 | Token 过期 |
| AUTH_002 | 401 | 无效 Token |
| AUTH_003 | 400 | 邮箱已注册 |
| TASK_001 | 404 | 任务不存在 |
| TASK_002 | 403 | 无权操作该任务 |
| AI_001 | 503 | AI 服务暂不可用 |
| AI_002 | 422 | 无法解析输入内容 |

### 全局异常过滤器

```typescript
// 统一错误响应格式
{
  "code": "TASK_001",
  "message": "任务不存在",
  "timestamp": "2026-04-20T01:30:00Z",
  "path": "/api/v1/tasks/xxx"
}
```
