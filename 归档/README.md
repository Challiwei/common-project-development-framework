# 归档目录

> 本目录用于存放历史版本的文档归档。

## 使用说明

当步骤 6（一致性校验）不通过且老板决定重新整理时，按以下规则归档：

1. 创建子目录：`vN-项目简称/`（如 `v1-智能待办/`）
2. 将当前版本的以下文件复制到归档目录：
   - 初步想法
   - 完整想法
   - 产品需求文档
   - 校验记录
3. 在归档文件中标注归档信息（见 Front Matter）
4. 在工作目录中开始新版本

详细规范请参考 [[00-流程与规范/版本管理规范]]。
