## 1. 产品概述

### 1.1 产品背景

现有待办工具以"被动记录"为主，用户需手动设置优先级和提醒。随着 AI 技术成熟，市场需要一款能"主动理解"用户任务的智能工具。目标用户（忙碌的知识工作者）有强烈的效率优化需求。

### 1.2 产品目标

- **目标1**：上线后 3 个月内获取 10,000 注册用户
- **目标2**：用户日活跃率（DAU/MAU）达到 40%
- **目标3**：AI 优先级推荐的用户采纳率达到 70%

### 1.3 目标用户

- **主要用户**：25-45 岁的创业者、管理者、自由职业者
- **使用场景**：移动端碎片时间管理任务
- **典型特征**：每日处理 10+ 待办事项，需要高效的优先级管理

### 2.2 功能详细描述

#### FR-001：自然语言任务创建

- **描述**：用户输入自然语言文本，系统自动解析为结构化的待办事项
- **用户故事**：作为 忙碌的创业者，我希望 用一句话快速创建任务，以便 不打断当前工作流
- **前置条件**：用户已登录
- **正常流程**：
  1. 用户点击 "+" 或在输入框输入文字
  2. 输入类似 "下周五之前跟供应商确认报价" 的自然语言
  3. 系统自动解析出：任务内容（跟供应商确认报价）、截止日期（下周五）
  4. 显示解析结果，用户确认或修改
  5. 任务创建成功
- **异常流程**：
  1. 系统无法解析日期 → 提示用户手动选择截止日期
  2. 输入内容过短或无意义 → 提示用户补充信息
- **验收标准**：
  - [x] 自然语言输入后 2 秒内完成解析
  - [x] 日期解析准确率 ≥ 90%
  - [x] 支持中文和英文输入

#### FR-002：AI 优先级分析

- **描述**：系统根据多维度信息自动为任务设置优先级（高/中/低）
- **用户故事**：作为 任务繁多的管理者，我希望 系统自动判断哪些任务更紧急，以便 合理分配时间
- **前置条件**：任务已创建，有足够的上下文信息
- **正常流程**：
  1. 新任务创建后，AI 自动分析
  2. 综合考虑：截止日期、任务关键词、历史行为模式
  3. 自动设置优先级标签（高/中/低）
  4. 用户可手动覆盖 AI 判断
- **异常流程**：
  1. 信息不足以判断 → 默认设为"中"，提示用户确认
- **验收标准**：
  - [x] AI 优先级推荐采纳率 ≥ 70%
  - [x] 用户手动覆盖后，AI 学习并调整后续推荐
  - [x] 新用户冷启动时，基于通用规则给出合理推荐

#### FR-003：今日推荐清单

- **描述**：每天早上基于 AI 分析生成当日推荐任务列表
- **用户故事**：作为 自由职业者，我希望 早上打开 App 就知道今天该做什么，以便 快速进入工作状态
- **前置条件**：用户有未完成的任务
- **正常流程**：
  1. 用户打开 App（或在设定的时间自动推送）
  2. 系统展示 "今日推荐" 清单（3-7 个任务）
  3. 清单按优先级排序，附带简要理由
  4. 用户可以接受、调整或忽略推荐
- **异常流程**：
  1. 没有待完成任务 → 显示 "今日无待办，享受轻松吧 🎉"
- **验收标准**：
  - [x] 每日推荐清单在 App 打开后 1 秒内加载
  - [x] 推荐任务数量控制在 3-7 个
  - [x] 每个推荐附带 AI 推荐理由（一句话）

## 4. 数据需求

### 4.1 核心数据实体

| 实体 | 关键字段 | 说明 |
|------|----------|------|
| User | id, email, name, created_at | 用户账号信息 |
| Task | id, user_id, title, description, due_date, priority, status | 待办任务 |
| AIAnalysis | id, task_id, priority_score, reasoning | AI 分析结果 |
| Reminder | id, task_id, trigger_type, trigger_condition | 提醒规则 |

### 4.2 数据存储与隐私

- **存储方式**：云端存储（PostgreSQL + Redis 缓存）
- **数据备份**：每日自动备份，保留 30 天
- **隐私合规**：GDPR 合规，用户可导出和删除个人数据
