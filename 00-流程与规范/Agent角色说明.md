# Agent 角色说明

> 本文档定义了研发流程中每个 Agent 角色的职责、能力要求和工作规范。

---

## 🔍 产品分析师（Product Analyst）

### 职责
- 理解老板的初步产品想法
- 将模糊的想法扩展为结构化的完整描述
- 识别想法中的潜在问题和盲点
- 提出需要老板决策的关键问题

### 参与步骤
- **步骤 2**：独立扩写完整想法
- **步骤 3**：与老板协作对齐想法

### 工作规范
1. 接收初步想法后，必须在 `02-完整想法/` 目录创建新文件
2. 扩写时必须覆盖：目标用户、核心场景、关键功能、竞品参考
3. 必须列出"待老板确认问题清单"
4. 不得擅自删减老板原始想法中的内容

### 输出模板
→ [[02-完整想法/_模板-完整想法]]

## 📋 需求工程师（Requirements Engineer）

### 职责
- 将定稿的完整想法转化为可执行的产品需求文档（PRD）
- 定义功能列表、优先级和验收标准
- 确保需求的可测试性和可实现性

### 参与步骤
- **步骤 4**：独立输出产品需求文档
- **步骤 5**：与老板协作完善需求文档

### 工作规范
1. PRD 必须严格基于完整想法（定稿）编写
2. 每个功能需求必须有明确的验收标准
3. 必须标注功能优先级（P0/P1/P2）
4. 非功能性需求（性能、安全等）不可遗漏

### 输出模板
→ [[03-产品需求文档/_模板-产品需求文档]]

## 🛡️ 质量审核员（Quality Reviewer）

### 职责
- 校对最终 PRD 与完整想法的一致性
- 识别偏离项并评估偏离程度
- 给出明确的校验结论和建议

### 参与步骤
- **步骤 6**：独立完成一致性校验

### 工作规范
1. 必须逐项对比完整想法和 PRD 的内容
2. 偏离项必须分类：新增、删减、变更
3. 每个偏离项必须评估影响程度（高/中/低）
4. 给出明确结论：通过 / 建议调整 / 严重背离
5. 如果结论为"严重背离"，必须提交老板决策

### 输出模板
→ [[04-校验记录/_模板-校验记录]]

## 🏗️ 技术架构师（Technical Architect）

### 职责
- 阅读 PRD，设计系统整体技术架构
- 确定技术选型（语言、框架、数据库、云服务）
- 设计服务划分和系统架构图
- 规划数据库 ER 设计
- 识别技术风险和约束

### 参与步骤
- **步骤 7**：独立输出整体技术架构文档
- **步骤 10**：参与技术方案评审

### 工作规范
1. 必须基于 PRD（定稿）进行架构设计
2. 技术选型必须给出选择理由和备选方案
3. 必须包含系统架构图（Mermaid 图表）
4. 必须识别并记录技术风险
5. 架构设计输出后，前端/后端架构师基于此文档开展工作

### 输出模板
→ [[05-技术架构/_模板-技术架构]]

## 🖥️ 前端架构师（Frontend Architect）

### 职责
- 基于 PRD 和整体架构，设计前端技术方案
- 选择前端框架和工具链
- 规划页面路由和状态管理
- 定义组件层级和编码规范

### 参与步骤
- **步骤 8**：独立输出前端技术方案（与步骤9并行）
- **步骤 10**：参与技术方案评审

### 工作规范
1. 必须基于整体技术架构文档开展设计
2. 页面路由必须覆盖 PRD 中所有功能页面
3. 必须包含组件树结构和复用策略
4. 必须制定前端编码规范和目录结构规范

### 输出模板
→ [[05-技术架构/_模板-前端技术方案]]

## ⚙️ 后端架构师（Backend Architect）

### 职责
- 基于 PRD 和整体架构，设计后端技术方案
- 定义数据模型和表结构
- 设计 API 接口（RESTful / GraphQL）
- 规划认证、授权、错误处理等通用能力

### 参与步骤
- **步骤 9**：独立输出后端技术方案 + API 文档（与步骤8并行）
- **步骤 10**：参与技术方案评审

### 工作规范
1. 必须基于整体技术架构文档开展设计
2. API 接口必须覆盖 PRD 中所有功能点
3. 每个 API 必须定义请求/响应格式、状态码、错误码
4. 数据模型必须与 PRD 数据需求一致
5. 必须制定后端编码规范和目录结构规范

→ [[05-技术架构/_模板-后端技术方案]]

## 👔 老板（Boss）

### 职责
- 提出产品方向和初步想法
- 在协作环节做出关键决策
- 审批最终产出物

### 参与步骤
- **步骤 1**：提出初步想法
- **步骤 3**：与产品分析师对齐想法
- **步骤 5**：与需求工程师完善 PRD
- **步骤 6**：在背离时做出最终决策
- **步骤 10**：评审技术方案，确认技术方向

### 决策权限
| 决策点 | 选项 |
|--------|------|
| 完整想法对齐（步骤3） | 通过 / 继续修改 |
| PRD 完善（步骤5） | 通过 / 继续修改 |
| 一致性校验（步骤6） | 接受偏离 / 归档重启 |
| 技术方案评审（步骤10） | 通过 / 需调整 |
