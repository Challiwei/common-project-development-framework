## 5. 状态流转记录

| 时间 | 操作人 | 状态变更 | 说明 |
|------|--------|----------|------|
| | 🧪 测试工程师 | → 新建 | 发现缺陷 |
| | {对应总监} | → 已确认 | 确认为有效缺陷 |
| | {对应总监} | → 修复中 | 已分派修复任务 |
| | {修复人} | → 待回归 | 修复完成 |
| | 🧪 测试工程师 | → 已关闭 / → 重新打开 | 回归测试结果 |

> **状态流转说明**：
> - **新建** → **已确认**：对应板块总监确认缺陷有效
> - **已确认** → **修复中**：总监创建修复任务卡片并分派
> - **修复中** → **待回归**：修复完成，等待测试工程师回归验证
> - **待回归** → **已关闭**：回归测试通过
> - **待回归** → **重新打开**：回归测试未通过，需继续修复
> - 任何阶段 → **已拒绝**：经确认非缺陷（设计如此 / 无法复现）

> **严重程度说明**：
> - **致命**：系统崩溃、数据丢失、核心功能完全不可用
> - **严重**：核心功能异常、数据错误、无变通方案
> - **一般**：非核心功能异常、有变通方案
> - **建议**：UI美观、体验优化、文案调整等
